# Change Classification

## Major

Breaking architectural changes.

Examples:

- repository restructuring;
- execution model redesign;
- governance redesign.

---

## Minor

New functionality without breaking compatibility.

Examples:

- new Execution Modules;
- new execution categories;
- documentation enhancements.

---

## Patch

Non-breaking improvements.

Examples:

- typo corrections;
- clarification;
- formatting;
- reference updates.
