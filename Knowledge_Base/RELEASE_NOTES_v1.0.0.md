**AI-OS** is a layered, platform-agnostic architecture standard for operating AI assistants as governed systems — predictable behavior, traceable changes, and token-efficient context instead of prompt chaos.

## Highlights

- **7-layer architecture** — Core identity, Architecture standards, Capability modules (11 domains), Execution processes (6 governed workflows), Platform adapters, Templates, and a 3-tier Memory model.
- **76 governed documents** — every document carries an immutable `DOC-<LAYER>-<NNN>` ID, semantic version, lifecycle status, and owner, tracked in a single Document Registry (SSOT).
- **Zero-defect CI gate** — `validate_aios.py` checks every push against the project's own governance rules (metadata, registry consistency, naming, terminology). The build fails on any regression.
- **Reasoning Patterns library (RP-001…010)** — Chain-of-Thought, Tree-of-Thoughts, Self-Consistency, ReAct with fallback budgets, context stacking, batch processing, inverted positioning, meta-prompting, constitutional critique, recursive decomposition — one authoritative catalog, referenced by ID everywhere.
- **AI-OS MCP Server** — a dependency-free, read-only Model Context Protocol server (`Knowledge_Base/aios_mcp_server.py`) that serves the registry and modules to any MCP client as on-demand tools. Sandboxed paths, no write surface, container isolation included (`Dockerfile.mcp`).
- **Working Kits** — deterministic single-file context builds: full (with provenance, for audit) and lean (token-efficient, for the model's context window), both regenerated by one script.
- **Claude Platform Adapter + production Prompt Generator Skill** — assembly rules mapping the platform-agnostic core onto Claude Projects / Claude Code / MCP.
- **Design Principles (DP-001…015) and 15-principle governance** — SSOT, DRY, Separation of Concerns, Platform-Agnostic Design, Quality by Design, Evidence-Based Decision Making.

## Getting Started

1. Read the [README](https://github.com/ASskerGarry/AI-Operating-Architecture-Standard#readme) — "Why AI-OS" and the layer map.
2. Deploy on Claude via the [Claude Adapter](04_Platforms/Claude_Adapter.md), or connect any MCP client to the [AI-OS MCP Server](04_Platforms/AIOS_MCP_Server.md).
3. Drop [`AI-OS_AI_Working_Kit_lean.md`](AI-OS_AI_Working_Kit_lean.md) into your model's context for the fastest start.
