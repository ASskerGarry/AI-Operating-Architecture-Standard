# 1. Purpose
The Capability Change Log maintains the complete historical record of all changes applied to Capability Modules.
Its purpose is to provide traceability, version history, impact assessment, and governance transparency across the Capability Layer.
---
# 2. Change Types
The following change types shall be recorded:
- Added
- Updated
- Improved
- Fixed
- Deprecated
- Removed
---
# 3. Required Information
Every change entry shall contain:
- Date
- Capability Module
- Version
- Change Type
- Description
- Reason
- Impact
- Author
---
# 4. Impact Classification
Each change shall be classified as:
- Low
- Medium
- High
- Critical

# 6. Governance Rules
Every published Capability Module update shall generate a corresponding Change Log entry.
Historical records shall never be deleted.
Corrections shall be recorded as new entries.
