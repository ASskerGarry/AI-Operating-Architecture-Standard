---
# 1. Purpose
This guide defines the official standard for writing, documenting, and maintaining Capability Modules within AI-OS.
Its purpose is to ensure consistency, clarity, maintainability, and interoperability across the Capability Library.
This guide governs documentation practices only.
---
# 2. Scope
This guide applies to every Capability Module within AI-OS.
All Capability documentation shall comply with this specification.
---
# 3. Writing Objectives
Capability documentation shall:
- describe one professional competency;
- remain technically accurate;
- be reusable;
- remain implementation-independent;
- support long-term maintenance.
---
# 4. Writing Rules
Capability documentation shall:
- use precise terminology;
- remain objective;
- avoid ambiguity;
- use technical language;
- describe expertise rather than execution.
Capability documentation shall never describe workflow logic.
---
# 5. Capability Content
Every Capability shall describe:
- professional expertise;
- supported activities;
- methodologies;
- best practices;
- limitations;
- expected deliverables.
Capabilities shall not define execution sequences.
---
# 6. Writing Style
Documentation shall:
- use concise language;
- use consistent terminology;
- remain implementation-neutral;
- avoid conversational wording;
- avoid subjective statements.
---
# 7. Terminology
Official terminology shall comply with the AI-OS Glossary.
Alternative terminology shall not be introduced without governance approval.
---
# 8. Validation
Before publication every Capability shall be reviewed for:
- completeness;
- consistency;
- correctness;
- clarity;
- maintainability;
- documentation quality.
---
# 9. Architectural Compliance
Capability documentation SHALL comply with:
- Core Identity
- Core Execution Engine
- AI-OS Architectural Principles
- Documentation Standards
This guide defines documentation conventions only.
Architectural principles SHALL NOT be duplicated within this document.
