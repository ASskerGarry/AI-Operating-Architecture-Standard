Version: 1.0.0
Status: Draft
Layer: Architecture
Document Type: Guide

---

## Purpose

This kit defines a standard pilot delivery structure for AI-OS Open-Core engagements.

It ensures pilot execution is consistent, measurable, and directly convertible into paid pack deployment.

---

## Scope

Included:

- pilot setup, execution cadence, and handoff format;
- minimum artifact set for pilot completion;
- KPI and retrospective structure.

Excluded:

- full long-term managed operations;
- platform-specific adapter implementation details;
- client legal onboarding steps.

---

## 1. Pilot Format

Recommended pilot duration: 2-4 weeks.

| Week | Focus | Required Output |
| ---- | ----- | --------------- |
| Week 1 | Setup and baseline | Current-state map and pilot success criteria |
| Week 2 | Controlled implementation | First governed workflow draft |
| Week 3 | Validation and iteration | Quality findings and updated artifacts |
| Week 4 | Readout and handoff | Pack activation recommendation and 30-day next plan |

---

## 2. Pilot Success Criteria

Pilot SHALL be considered successful when:

1. at least one workflow reaches governed release readiness;
2. key gaps are documented with evidence and priority;
3. client owners accept a post-pilot activation plan.

---

## 3. Required Pilot Artifacts

1. pilot charter and scope boundary;
2. baseline scorecard;
3. risk and issue log;
4. final recommendation memo;
5. next-step implementation plan (30 days).

---

## 4. Execution Cadence

- Weekly steering checkpoint (scope, risk, decisions).
- Mid-week working review (artifact quality and blockers).
- Final readout with commercial handoff recommendation.

All decisions SHALL be recorded in the pilot issue log.

---

## 5. Handoff to Paid Packs

| Pilot Outcome | Next Commercial Step |
| ------------- | -------------------- |
| Governance blockers dominate | Governance Pack activation |
| Prompt lifecycle blockers dominate | PromptOps Pack activation |
| Both blocker groups present | Combined Pack activation |

Handoff SHALL include owner, timeline, and first milestone definition.

---

## 6. Pilot KPI Dashboard

| KPI | Target |
| --- | ------ |
| Pilot completion on time | >= 80% |
| Stakeholder alignment score | >= 8/10 |
| Pilot-to-paid conversion | >= 30% |
| Time to post-pilot activation decision | <= 5 business days |
